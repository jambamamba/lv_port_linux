## Goals

## Summary

### Problem: Screenshot not generated
- `weston-screenshooter` fails with `unauthorized` (compositor restricts `weston_capture_v1`)
- Python path exits early because `lv_wayland_window_is_open()` returns false on first call
- Solution: Added a JSON-only code path in `main.cpp` that runs 20 LVGL event-ticks then calls `dumpScreenshot()`

### Key discovery: `WAYLAND_DISPLAY` must be set
- Weston runs on `wayland-1` socket but the env var isn't always set
- Without it, `lv_wayland_init` asserts and segfaults
- Fix: `export WAYLAND_DISPLAY=wayland-1`

### Fix: restaurant-1 image path resolution
- `LeleImageConverter::generateImgDsc()` resolves relative image paths against `std::filesystem::current_path()`
- When running from a different CWD, `foods/IMG_*.png` resolves to wrong path → `LL(FATAL)` → `abort()`
- Fix: `main.cpp` now calls `std::filesystem::current_path(json_dir)` after resolving the JSON path to absolute

- `apple-info.json` — screenshot captured (61 KB, 1024×768)
- `restaurant-1.json` — screenshot captured (717 KB, 1024×768), all 25 food images loaded
